# BC Surgical Wait Times — Analytics Pipeline

End-to-end data pipeline and analytical layer for British Columbia's publicly published quarterly surgical wait times dataset. Covers FY2009/10 through FY2024/25 at facility and procedure level across all six regional health authorities and PHSA.

---

## Background

BC's Ministry of Health publishes quarterly surgical wait time data under its commitment to public accountability for health system performance. The dataset records, for each combination of health authority, hospital, and procedure group, how many patients were on the wait list at quarter-end, how many surgeries were completed that quarter, and the 50th and 90th percentile wait times for completed cases.

This project builds a reproducible analytical pipeline on top of that data: cleaning, classifying data quality issues, deriving analytical flags, loading into a queryable database, and exposing pre-built SQL views for five specific policy and operational questions. The output drives a tiered Power BI dashboard designed for three distinct audience levels.

**Dataset scope:**

| Dimension | Coverage |
|-----------|----------|
| Fiscal years | FY2009/10 – FY2024/25 (16 years, quarterly) |
| Source rows | 202,953 |
| Rows after cleaning | ~155,000 (rollup rows removed) |
| Health authorities | 7 (Fraser, Interior, Northern, PHSA, Vancouver Coastal, Vancouver Island + provincial aggregate) |
| Hospitals | 66 |
| Procedure groups | 85 |

---

## Analytical Questions

The pipeline is structured around five specific questions. Each has a dedicated SQL view.

| # | Question | View |
|---|----------|------|
| Q1 | Is BC's surgical backlog recovering post-COVID, and how does the current quarter compare to the pre-COVID baseline? | `view_covid_recovery` |
| Q2 | Which health authorities have the longest median wait times, and is the gap between regions widening or narrowing? | `view_regional_wait_times` |
| Q3 | Which procedure types have the highest patient volumes currently waiting, and how has that changed over five years? | `view_procedure_volume_trends` |
| Q4 | Are high-volume procedures meeting BC's provincial wait time benchmarks (P50 ≤ 182 days, P90 ≤ 364 days)? | `view_benchmark_compliance` |
| Q5 | Which hospitals are statistical outliers against the provincial average for the same procedure and quarter? | `view_hospital_outliers` |

## Data Source

**Publisher:** BC Ministry of Health
**Dataset:** Surgical Wait Times — Quarterly
**File format:** Excel (.xlsx)
**Expected location:** `data/raw/2009_2025-quarterly-surgical_wait_times-final.xlsx`

The Ministry publishes updated files quarterly. When a new release is available, replace the source file and rerun the pipeline. The `docs/pipeline_runbook.md` documents the full update procedure including schema compatibility checks and constants to update.

### Source Columns

| Column | Type | Nullable | Description |
|--------|------|----------|-------------|
| `FISCAL_YEAR` | String | No | BC fiscal year label, e.g. "2023/24" (April–March) |
| `QUARTER` | String | No | Q1 (Apr–Jun), Q2 (Jul–Sep), Q3 (Oct–Dec), Q4 (Jan–Mar) |
| `HEALTH_AUTHORITY` | String | No | One of six regional HAs or provincial aggregate |
| `HOSPITAL_NAME` | String | No | Specific facility or "All Facilities" aggregate |
| `PROCEDURE_GROUP` | String | No | One of 85 surgical procedure categories |
| `WAITING` | Integer | No | Patients on the wait list at **quarter-end** (snapshot, not cumulative) |
| `COMPLETED` | Integer | No | Surgeries completed within the quarter |
| `PERCENTILE_COMP_50TH` | Float | Yes | Median days from wait-list entry to surgery — completed cases only |
| `PERCENTILE_COMP_90TH` | Float | Yes | 90th percentile days — completed cases only |

## Pipeline — Data Cleaning Decisions

All cleaning decisions are fully documented with reasoning and invalidating conditions in `docs/assumptions_log.md`. The summary below covers the ten key decisions.

| ID | Decision | Rationale |
|----|----------|-----------|
| A1 | Remove rollup rows using OR logic across all three sentinel values | Any partially-aggregated row causes double-counting if summed; AND logic would leave partial rollups in place |
| A2 | Treat percentile nulls (36.4%) as structural suppression, not errors | BC suppresses percentiles when completed cases < ~5; null rate is stable across all 16 years with no temporal drift |
| A3 | Flag 54 anomalous rows (`COMPLETED > 0` but percentile null) separately | These do not fit the suppression pattern and are concentrated in specific Northern Health procedures |
| A4 | COVID flag covers FY2020/21 and FY2021/22 only; Q4 2019/20 excluded | Flagging Q4 2019/20 would contaminate the 2018/19 pre-COVID baseline used throughout the analysis |
| A5 | FY2024/25 flagged provisional for the entire fiscal year | File was last modified March 8, 2025; BC routinely revises prior quarters in subsequent releases |
| A6 | "All Other Procedures" excluded from trend and compliance views | This residual category's apparent volume decline reflects reclassification into named groups over time, not real volume change |
| A7 | Volume-weighted average of P50 used as regional aggregate | A true median requires individual patient records; volume-weighted average is the closest valid aggregate with pre-computed percentiles |
| A8 | 2-sigma (z > 2.0) threshold for hospital outlier classification | Standard healthcare benchmarking convention; 1.5σ over-flags small-hospital variance, 3σ misses systemic issues |
| A9 | Percentile columns rounded to 1 decimal place | Corrects floating-point representation artifacts from Excel without discarding clinically relevant sub-day precision |
| A10 | High-volume procedures defined as top quartile by WAITING in FY2023/24 | Relative quartile threshold is stable as system volume changes; fixed count thresholds are not |

## Known Limitations

These limitations apply to all findings from this dataset. Full detail including disclosure requirements for each chart type is in `docs/limitations_disclosure.md`.

| # | Limitation | Practical Implication |
|---|------------|----------------------|
| L1 | P50/P90 measure **completed-case** wait times only | Patients still waiting are invisible; clearing short-wait cases first improves percentiles while long waiters accumulate |
| L2 | 36.4% of percentile values are suppressed | Regional averages over-represent high-volume facilities; true system-wide wait times are likely understated |
| L3 | Urgency mix not captured | Facilities that transfer urgent cases will show shorter waits on paper; cross-facility comparisons are descriptive only |
| L4 | COVID years (2020/21, 2021/22) are not comparable | These years reflect policy-driven deferrals, not system performance; trend lines should not cross this period without annotation |
| L5 | FY2024/25 is provisional | Q4 2024/25 was incomplete at file modification date (March 8, 2025); figures should be treated as preliminary |
| L6 | Weighted average is not a true statistical median | Regional/provincial P50 figures are approximations; they may differ from a true population median |
| L7 | Z-score outlier method assumes normality | Healthcare wait distributions are right-skewed; low-volume hospitals may appear as outliers due to statistical variance |
| L8 | No population denominator | Per-capita or age-standardised rates cannot be computed; volume comparisons across regions are descriptive only |
| L9 | No outcome data | This analysis measures access, not quality; shorter wait times are not necessarily better |
| L10 | Coding variation across facilities | Facilities may apply different conventions for wait-list entry dates and procedure classification |

## Derived Columns

The pipeline adds seven columns to the fact table. These do not exist in the source file.

| Column | Type | Derivation |
|--------|------|------------|
| `fiscal_year_start` | Integer | First four characters of `FISCAL_YEAR` cast to integer. "2023/24" → 2023 |
| `quarter_number` | Integer | Q1→1, Q2→2, Q3→3, Q4→4 |
| `period_sort_key` | Integer | `fiscal_year_start × 10 + quarter_number`. "2023/24 Q3" → 20233. Enables correct chronological ordering |
| `covid_flag` | Boolean | `TRUE` where `fiscal_year_start` ∈ {2020, 2021} |
| `provisional_flag` | Boolean | `TRUE` where `FISCAL_YEAR = "2024/25"` |
| `percentile_suppressed` | Boolean | `TRUE` where `PERCENTILE_COMP_50TH IS NULL`. Structural suppression by BC Ministry — not imputed |
| `anomalous_percentile` | Boolean | `TRUE` where `COMPLETED > 0 AND PERCENTILE_COMP_50TH IS NULL`. Distinct from structural suppression |

## SQL Views Reference

All views operate on `fact_surgical_waits`. Views that use percentile data filter `percentile_suppressed = FALSE` before aggregating.

---

### `view_covid_recovery`

**Question:** Is BC's surgical backlog recovering post-COVID?
**Granularity:** Province-wide, quarterly
**Filters:** None (volume-only view; includes all rows including those with suppressed percentiles)

| Column | Description |
|--------|-------------|
| `fiscal_year`, `quarter`, `period_sort_key` | Time dimensions |
| `covid_flag`, `provisional_flag` | Pass-through flags for visual filtering |
| `total_waiting` | Province-wide sum of WAITING for that quarter |
| `total_completed` | Province-wide sum of COMPLETED |
| `completion_rate_pct` | `total_completed / (total_waiting + total_completed) × 100` |
| `baseline_avg_waiting_2018_19` | Average quarterly WAITING in FY2018/19 (repeated on every row as reference) |
| `waiting_vs_baseline_pct` | `(total_waiting − baseline) / baseline × 100`. Positive = above pre-COVID baseline |

---

### `view_regional_wait_times`

**Question:** Which health authorities have the longest median wait times, and is the gap widening?
**Granularity:** Health authority × quarter
**Filters:** `percentile_suppressed = FALSE`; excludes "All Other Procedures"

| Column | Description |
|--------|-------------|
| `health_authority`, `fiscal_year`, `quarter`, `period_sort_key` | Dimensions |
| `weighted_avg_p50` / `weighted_avg_p90` | Volume-weighted average of facility P50/P90 (days) |
| `weighted_avg_p50_weeks` / `weighted_avg_p90_weeks` | Same values in weeks |
| `provincial_p50` / `provincial_p90` | Provincial benchmark for same quarter |
| `gap_vs_provincial_p50_days` | Positive = longer waits than provincial average |

**Caveat:** Weighted average of P50 is not a true statistical median (see L6 above).

---

### `view_procedure_volume_trends`

**Question:** Which procedure types have the most patients waiting, and how has that changed over five years?
**Granularity:** Procedure group × fiscal year (annual)
**Filters:** Excludes "All Other Procedures" (composition drift, see A6)

| Column | Description |
|--------|-------------|
| `procedure_group`, `fiscal_year`, `fiscal_year_start` | Dimensions |
| `annual_waiting` / `annual_completed` | Annual totals (sum across all four quarters) |
| `waiting_5yr_ago` | Waiting volume for same procedure five fiscal years prior (NULL if unavailable) |
| `waiting_5yr_change_abs` / `waiting_5yr_change_pct` | Absolute and percentage change |
| `rank_by_waiting` | Procedure rank within fiscal year by waiting volume (1 = highest volume) |

---

### `view_benchmark_compliance`

**Question:** Are high-volume procedures meeting BC's wait time targets?
**Granularity:** Procedure × health authority × quarter
**Filters:** `percentile_suppressed = FALSE`; top-quartile procedures by WAITING in FY2023/24 only
**Benchmarks:** P50 ≤ 182 days (26 weeks), P90 ≤ 364 days (52 weeks)

| Column | Description |
|--------|-------------|
| `procedure_group`, `health_authority`, period columns | Dimensions |
| `weighted_avg_p50` / `weighted_avg_p90` | Volume-weighted average P50/P90 (days) |
| `p50_threshold_days` / `p90_threshold_days` | 182 / 364 — BC provincial targets |
| `p50_compliant` | `TRUE` if weighted P50 ≤ 182 days |
| `p90_compliant` | `TRUE` if weighted P90 ≤ 364 days |
| `fully_compliant` | `TRUE` if both thresholds met; `NULL` if either percentile unavailable |

**Scope note:** Low-volume procedures are excluded (see A10). One anomalous case would dominate compliance rates for rare procedures.

---

### `view_hospital_outliers`

**Question:** Which hospitals are statistical outliers versus the provincial average for the same procedure and quarter?
**Granularity:** Hospital × procedure × quarter
**Filters:** `percentile_suppressed = FALSE`; excludes "All Other Procedures"

| Column | Description |
|--------|-------------|
| `hospital_name`, `health_authority`, `procedure_group`, period columns | Dimensions |
| `hospital_p50` / `hospital_p90` | Hospital-level volume-weighted average P50/P90 |
| `provincial_avg_p50` / `provincial_std_p50` | Provincial mean and standard deviation for same procedure + quarter |
| `deviation_from_avg_pct` | `(hospital_p50 − provincial_avg) / provincial_avg × 100` |
| `z_score_p50` | Standard deviations from provincial mean. `NULL` where standard deviation = 0 |
| `outlier_flag` | `TRUE` where `|z_score_p50| > 2.0` |
| `outlier_direction` | "high", "low", or "within_range" |
| `total_waiting` | Apply a minimum volume filter before interpreting outlier flags |

**Caveat:** Always filter by `total_waiting > N` before presenting outlier results. Small-volume hospitals generate extreme z-scores due to statistical variance, not systemic performance differences (see L7).

## Power BI Dashboard

**File:** `powerbi/Healthline.pbit` (Power BI template)

The dashboard is organized into three audience tiers:

| Tier | Audience | Focus |
|------|----------|-------|
| 1 — Executive | Health authority leadership, ministry | Province-wide backlog trend, COVID recovery status, benchmark compliance rate |
| 2 — Policy | Health authority planning teams | Regional comparisons, procedure-level volume trends, gap-vs-provincial analysis |
| 3 — Operational | Hospital managers, surgical program leads | Hospital outlier identification, procedure-level compliance, quarter-by-quarter throughput |

### Connecting to the DuckDB file

The dashboard reads directly from the DuckDB file generated by the pipeline.

1. Install the [DuckDB ODBC driver](https://duckdb.org/docs/api/odbc/overview)
2. Open `powerbi/Healthline.pbit` in Power BI Desktop
3. When prompted for the data source path, provide the absolute path to `data/processed/surgical_wait_times.duckdb`
4. Refresh all queries

After the pipeline runs and regenerates the DuckDB file, click **Refresh** in Power BI Desktop to pull updated data.

## How to Reproduce

### 1. Install Python dependencies

```bash
pip install -r requirements.txt
```

Requires Python 3.10 or later.

### 2. Obtain the source data file

Download the latest quarterly surgical wait times Excel file from BC's Ministry of Health and place it at:

```
data/raw/2009_2025-quarterly-surgical_wait_times-final.xlsx
```

The `data/` directory is not tracked in version control — create it locally.

### 3. Run the pipeline

```bash
python -m pipeline.run_pipeline
```

The pipeline will:
- Load the Excel file and report raw row count
- Remove pre-aggregated rollup rows (down to ~155,000 rows)
- Classify percentile null patterns and derive analytical flags
- Export the cleaned dataset to `data/interim/surgical_wait_times_cleaned.csv`
- Create the DuckDB database at `data/processed/surgical_wait_times.duckdb`
- Execute all five SQL view definitions

### 4. Run SQL validation checks (optional)

```bash
duckdb data/processed/surgical_wait_times.duckdb
```

Then run queries against any of the five views to spot-check results:

```sql
-- Check province-wide quarterly totals
SELECT fiscal_year, quarter, total_waiting, total_completed
FROM view_covid_recovery
ORDER BY period_sort_key;

-- Check regional wait time gaps
SELECT health_authority, fiscal_year, quarter, weighted_avg_p50, gap_vs_provincial_p50_days
FROM view_regional_wait_times
WHERE fiscal_year = '2023/24'
ORDER BY gap_vs_provincial_p50_days DESC;
```

### 5. Open Power BI

Open `powerbi/Healthline.pbit`, point the data source at `data/processed/surgical_wait_times.duckdb`, and refresh.

## Updating for New Quarterly Releases

When BC publishes a new quarterly data file:

1. Replace the file in `data/raw/` with the new release
2. Verify the source schema still has the expected nine columns
3. Update `PROVISIONAL_FISCAL_YEAR` in the pipeline if a new fiscal year has become final
4. Rerun `python -m pipeline.run_pipeline`
5. Refresh Power BI

Full update procedure including schema checks, validation queries, and troubleshooting steps is documented in `docs/pipeline_runbook.md`.

## Benchmark Reference

BC's provincial surgical wait time targets:

| Metric | Threshold |
|--------|-----------|
| P50 (median) | ≤ 182 days (26 weeks) |
| P90 (90th percentile) | ≤ 364 days (52 weeks) |

These targets apply to completed-case wait times and are the basis for the compliance flags in `view_benchmark_compliance`.
